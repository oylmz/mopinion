Computer says: 'no!'

1. Unknown / Generic error

Our servers are ok, the request was accepted and appears to be valid.
There might be some additional information about what happened in the response body.
If not, check your request carefully, and try again. If you keep getting this error, drop us a line

2. Server error

It's not you,... it's us.
Unfortunately we can't give you any more information at this point. The good news is you can try your request again. Just not too soon. Go get a coffee first.

3. Public key not found

Looks like the public key you provided is incorrect. Better make sure you didn't make a typo.

4. Invalid Token

Looks like the token you provided in the x-auth-token header is incorrect.

5. Invalid JSON

The request body you send is not properly formatted JSON.
Go check for typo's, or check your data on jsonlint.com

6. Route does not exist

The resource you are requesting is not available,.. or at least not at this endpoint.

7. Report id is not set

You need to provide a report-id.
Use the /account endpoint to see valid id's

8. Report not found

The report-id you provided does not point to a report in your account.
Use the /account endpoint to see valid report id's

9. Failed to update the resource

Your requests appears to be ok, but something went wrong trying to process it.
You may try the request again.

10. You don't have enough rights for this action

Your requests appears to be ok, but you're just not allowed to perform whatever it was you tried to do.

17. There are no deployment codes for your account

18. The credentials you provided are not valid

The resource you are trying to reach, need credentials in the form of an Authorization header. (Not an 'x-auth-token' header with a signature token)
You can create one using your public key and private key following the RFC1945 -11.1 Basic Authentication Scheme.

19. There are no results for the page number you requested

The resource-collection does exist, but the page number you provided exceeds the number of pages available.

20. Deployment id is not set

You need to provide a deployment-id.

21. Deployment not found

We can find a deployment for the deployment-id you provided.

22. Organisation not found

There is no organisation with the id you provided.

23. Account not found

The credentials / signature are valid, but it looks like there's no data associated with them.
This is very unusual... Better contact us
